## 一、产品概述

### 1.1 产品背景

目前市面上大量儿童英语学习App以AI生成或改编内容为主，语言"塑料感"较重，缺乏真实语料的自然语感；同时，"听说输出"类训练（尤其是复述、口译）几乎是空白地带，多数产品仍停留在跟读打卡和选择题测验层面。

本产品创始人具备香港中文大学翻译系背景、新东方口译及托福雅思教学经验，并作为8岁孩子的母亲亲历儿童英语启蒙全过程，兼具语言教学专业能力、儿童认知理解与内容运营经验（小红书双语育儿账号），具备打造差异化产品的独特优势。

### 1.2 产品定位

以 YouTube 优质原版英语内容为素材库，面向 5–12 岁儿童，提供"视频精学 + 互动测验 + 少儿口译练习"一体化的英语学习网站，强调真实语料输入与语言输出能力并重。

**产品名："小译星" LittleVoicer**
以"小小译者、开口发光"为品牌意象，星星既是IP形象也是激励体系（星数）的天然延伸，便于后续设计吉祥物、勋章与小红书传播视觉。

### 1.3 目标用户

- **核心使用者**：5–12岁儿童，划分为 L1 启蒙段（5–7岁）、L2 提升段（8–10岁）、L3 进阶/口译段（11–12岁）
- **决策及陪伴者**：重视英语启蒙质量的家长，尤其是双语家庭、留学规划家庭
- **用户共性**：已具备一定英语基础，追求原汁原味语言输入，不满足于应试型刷题
- **注册方式**：家长注册制。家长使用邮箱注册账号，可为多个孩子创建子档案，管理各自的学习进度和口译录音

### 1.4 核心价值主张

- **内容真实**：直接取材 YouTube 母语者原创内容，而非 AI 生成或改编文本
- **分级科学**：按年龄认知特点 + 语言难度双轨分级
- **输出导向**：不止于听读输入，增设复述、少儿口译等语言产出训练，形成市场差异化
- **游戏化驱动**：闯关地图、星星勋章体系提升儿童自主学习动力
- **无障碍访问**：通过镜像/备用源方案，确保大陆家长和孩子无需翻墙即可流畅使用

## 二、内容策略：YouTube 英文教育资源库建设

### 2.1 内容来源与筛选标准

- **优先频道类型**：科普类（自然/动物/生活常识向儿童科普频道）、故事绘本朗读类、自然拼读与语音启蒙类、儿童生活情景类
- **筛选标准**：发音清晰标准、语速适中、原生自带或可生成字幕、单集时长3–8分钟为宜、内容积极正向、无广告或不当内容
- **推荐频道举例**：
  - 科普类：National Geographic Kids、SciShow Kids、Peekaboo Timetube
  - 故事类：Oxford Owl、Super Simple Songs、Starfall
  - 自然拼读：Jolly Phonics、Alphablocks
  - 生活情景：Ms Rachel (Songs for Littles)、Art for Kids Hub

### 2.2 内容分类体系

- **按主题**：自然科学、动物世界、日常生活、故事绘本、社会认知
- **按难度**：L1 / L2 / L3 三级，对应不同年龄段词汇量与句式复杂度
- **按训练功能标签**：适合跟读、适合复述、适合口译
- **按访问源**：每个视频条目标记首选源（YouTube）和备用源（镜像站ID）

### 2.3 大陆访问方案（新增，V2.0 重点）

#### 问题背景

YouTube 在中国大陆无法直接访问，而目标用户中大量家长和孩子位于大陆地区。如果视频无法播放，整个产品价值归零。

#### 方案设计：双源播放 + 自动降级

| 层级 | 方案 | 适用场景 | 说明 |
|---|---|---|---|
| 源1 | YouTube 官方 IFrame 嵌入 | 海外用户、有网络条件的用户 | 默认首选源，画质最高、最及时 |
| 源2 | 国内镜像站/缓存站 | 大陆用户 | 通过第三方镜像服务（如 y2u.be、ytbcdn.com 等）加载同一视频，无需翻墙 |
| 降级 | 自动检测 + 手动切换 | 所有用户 | 前端自动检测 YouTube 可访问性，不可用时提示切换镜像源；同时在UI提供一键切换按钮 |

#### 技术实现

1. **镜像站硬编码模板**：在代码中预置一组镜像站URL模板（如 `y2u.be`、`vidplay.me` 等），前端自动轮换尝试直到找到一个能播放的。镜像站列表可通过环境变量 `NEXT_PUBLIC_MIRROR_SITES` 配置，方便随时增删而不需发版。

2. **前端自动检测**：页面加载时发起一个轻量级的 YouTube IFrame 加载测试（监听 `onReady` / `onError` 回调），若超时未就绪则自动标记为"不可用"
2. **镜像站切换**：当检测到 YouTube 不可用时，弹出温和提示"您所在地区可能无法访问 YouTube，点击下方按钮切换到备用播放源"，用户点击后播放器自动切换至镜像站URL
3. **视频元数据缓存**：视频封面、标题、频道名等元数据由后端通过 YouTube Data API v3 获取并缓存，即使视频本身无法播放，卡片展示不受影响
4. **备用播放器兜底**：对于部分镜像站不支持的视频，可在视频精学页面提供"复制链接到本地播放器"的引导文案，辅助高年级家长自行解决

#### 用户体验流程

```
打开视频精学页
  → 自动检测 YouTube 可达性
    → 可达：正常播放
    → 不可达：显示"切换备用源"按钮 → 用户点击 → 镜像站播放
```

#### 注意事项

- 镜像站并非官方服务，稳定性取决于第三方运营方，需在产品中做好降级预案
- 不自行搭建YouTube代理服务（法律风险过高），仅利用公开可用的镜像站
- 海外用户不受影响，保持完整体验

### 2.4 版权与合规说明

- **不下载、不重新分发原始视频文件**，统一采用 YouTube 官方 IFrame Player 嵌入播放，避免直接侵权风险
- **字幕与讲解文本自主原创整理**（听打+改写+配题），不直接爬取、翻译、转售 YouTube 官方字幕作为商用素材
- **产品条款中注明**"内容版权归原创作者/频道所有，本站仅作教育整理与引导跳转"，并保留频道来源标注
- **涉及付费会员环节需谨慎**，建议前期以免费内容浏览 + 自建练习题作为主要付费点，规避直接商业化引流原始视频的版权风险，必要时咨询专业法律意见
- **YouTube 嵌入播放的法律依据**：YouTube ToS 允许通过 IFrame Player API 嵌入视频，只要不修改播放器外观、不绕过广告、不剥离来源信息，嵌入行为本身不构成侵权（参考 YouTube Terms of Service §IV.B）

## 三、核心功能模块设计

### 3.1 首页 · 学习地图

- 卡通闯关地图，每个关卡对应一个 YouTube 精选视频/主题单元
- 已完成关卡点亮、当前关卡高亮、未解锁关卡置灰锁定
- 顶部展示连续打卡天数、累计星数、当前等级徽章

### 3.2 视频精学模块

- **视频源**：YouTube IFrame 嵌入播放器（支持 0.75x / 1x 倍速切换），内置双源切换机制（见2.3）
- **配套内容**：中英对照字幕（自建原创）、生词点击高亮查词、关键句型提炼卡片
- **访问状态指示**：播放器右上角显示当前播放源图标（YouTube / 镜像站），用户可随时切换

### 3.3 词汇卡片模块

- 从当集视频中提取核心词汇，图片+单词+音标+例句一体展示
- 支持翻转自测、滑动切换、一键加入个人生词库复习

### 3.4 互动测验模块

- 视频内容理解题（选择/判断）、听力填空题
- 即时反馈解析 + 自动生成错题本，支持二次巩固练习

### 3.5 少儿口译练习模块（核心特色，新增）

依托创始人口译教学专业背景设计的三级进阶体系，是本产品区别于市面同类竞品的核心壁垒。**评分方式第一阶段采用家长/人工评价**，暂不接入语音识别自动评分，先验证儿童使用体验与内容合理性。

**分级任务设计**

- **L1 跟读复述（5–7岁）**：听后模仿跟读单句，训练语音语调
- **L2 信息复述（8–10岁）**：听完一段话后用自己的话进行中文或英文复述大意，训练信息抓取能力
- **L3 情景交传（11–12岁）**：设计"小小导游""国际小记者"等仿真情景任务，听一段英文/中文后限时口译成另一种语言

**人工评分工作流（第一阶段）**

1. **录音采集**：儿童在任务页完成录音，系统自动保存并与原始示范音频配对
2. **回放对比**：家长端可一键切换播放"孩子录音 ↔ 示范音频"，便于直观对比语速、停顿与用词
3. **三维打分**：家长按以下维度对照评分表进行评价，每项采用三档快速打分（★☆☆ 待提升 / ★★☆ 良好 / ★★★ 优秀），降低操作门槛：
   - **完整度**：是否覆盖原文关键信息点
   - **流畅度**：是否有明显卡顿、重复、长时间停顿
   - **准确度**：用词/译文是否准确传达原意
4. **文字点评（可选）**：家长可附加一句鼓励性文字反馈，替代生硬分数，保护儿童学习积极性
5. **记录归档**：每次打分自动计入"口译成长记录"，按周/月生成评分趋势曲线，沉淀为家长学习报告的核心数据源
6. **儿童侧反馈**：孩子端仅展示星星与鼓励语，不直接暴露具体分数，避免打击自信心
7. **后期迭代预留接口**：数据结构预先支持接入语音识别转写结果，为后续自动评分（第三阶段）留出扩展空间，无需重构

### 3.6 复述与配音打卡模块

- "每日一句"跟读打卡，强化坚持性学习习惯
- 视频片段配音挑战，提升语音语调模仿趣味性

### 3.7 成长激励系统

- 星星/勋章体系，连续打卡奖励，阶段性成长证书
- 支持一键生成学习成果分享卡片，便于家长在小红书等社交平台分享，形成自然传播

### 3.8 家长后台

- 学习报告：学习时长、测验正确率、口译评分趋势可视化
- 口译打分入口：录音回放对比 + 三维打分 + 文字点评（见3.5）
- 内容与时长管控：家长可设置每日学习时长上限、筛选可见内容分类
- 周报导出功能，便于分享与留存

## 七、内容管理方案（新增，V2.0 重点）

### 7.1 为什么需要 Airtable / Notion

产品核心是内容——视频的选择、分级、配套素材的组织。独立开发者一个人完成内容筛选、字幕整理、题目编写的工作量很大，需要一个**低门槛、结构化、易维护**的内容管理工具。

Airtable / Notion 的优势：
- **零代码**：像填表格一样管理内容，不需要写任何代码
- **实时同步**：网站自动拉取最新数据，更新资源只需在 Airtable/Notion 中操作
- **结构化**：天然支持分类、标签、附件（封面图、音频文件等）
- **协作友好**：未来若有合作者帮忙整理内容，共享表格即可

### 7.3 前后端同步机制

```
Airtable/Notion（内容录入）
  ↓ REST API 定时拉取（每15分钟）或 webhook 实时触发
Next.js API Route（数据中转）
  ↓ 缓存 + 格式化处理
前端页面（展示给用户）
```

- **定时拉取**：Next.js 的 Edge Cron 或 Vercel Cron 每分钟/每小时调用 Airtable API 同步最新数据
- **Webhook**：Airtable Webhooks 功能，当某条记录变更时自动通知 Next.js 刷新缓存
- **缓存策略**：API 层对同步结果做短期缓存（TTL 5-15分钟），避免频繁请求 API
- **增量同步**：利用 Airtable 的 `createdTime` 字段实现增量更新，只拉取变更过的记录

### 7.4 资源更新工作流（管理员视角）

```
1. 发现优质视频 → 复制 YouTube 链接
2. 在 Airtable Videos 表中新增一行 → 粘贴链接 + 填写基本信息
3. 系统自动抓取：标题、频道名、封面图、时长（通过 YouTube Data API）
4. 人工补充：Level、Theme、Tags、Notes
5. 点击 Status 为"已上线" → 网站自动展示
6. 配套素材（词汇卡、测验题、口译任务）在对应表中逐条添加
7. 完成后前端页面实时更新，无需部署
```

**关键点**：更新资源 = 在 Airtable 中填一行表格，零代码、零部署、即时生效。

## 八、技术实现建议（面向 Vibe Coding 独立开发）

- **前端**：Next.js / React + Tailwind CSS，移动端优先的响应式设计
- **视频播放**：YouTube IFrame Player API，实现嵌入播放、进度监听、倍速控制
- **双源切换**：前端自动检测 YouTube 可达性 + 镜像站URL替换逻辑（见2.3）
- **用户系统与数据库**：Supabase（自带 Auth + Postgres + Storage），适合个人快速搭建与迭代
- **家长注册制**：家长邮箱注册 → 创建子档案（绑定孩子信息：姓名/年龄/等级）→ 各孩子独立学习数据
- **用户档案模型**：Supabase `profiles` 表（家长信息）+ `children_profiles` 表（孩子信息，关联家长）+ `learning_records` 表（学习记录，关联孩子）
- **口译录音**：浏览器 MediaRecorder API 采集录音，存入 Supabase Storage，与示范音频做配对存储
- **口译评分（第一阶段）**：仅需简单表单（三维星级 + 文字点评），写入数据库，无需接入第三方语音识别，降低初期开发成本
- **内容中台**：Airtable 作为内容管理后台（Videos / Vocabulary / Quizzes / Interpreting Tasks 四张表），通过 REST API 与 Next.js 同步
- **部署**：Vercel 海外节点，支持快速迭代与预览环境，内置 Edge Cron 用于定时同步
- **大陆访问优化**：Vercel 海外节点在大陆访问速度可能较慢，但核心视频通过镜像站加载不受影响；静态资源（HTML/CSS/JS）可通过 Cloudflare CDN 加速（可选升级方案）

## 十、风险与合规提示

### 10.1 版权风险

- 详见第二章内容合规说明，核心原则是"嵌入播放不下载、原创配套不转售"
- YouTube 嵌入播放符合 YouTube ToS，但需注意：不修改播放器外观、不绕过广告、不剥离来源信息
- 如未来引入付费体系，建议咨询专业律师评估嵌入播放模式的合规边界

### 10.2 大陆访问风险

- 镜像站非官方服务，稳定性依赖第三方运营方，存在随时失效的可能
- 应对策略：定期巡检视频可用性；对失效视频优先寻找同频道其他镜像源；在用户端做好降级提示
- **不自行搭建代理服务**（法律风险过高），仅利用公开可用的镜像站

### 10.3 内容时效性

- YouTube 内容可能被原作者删除或设为私有，需建立定期巡检机制，及时替换失效链接
- 建议在前端对每个视频卡片做"健康检查"：播放时若视频不存在，自动标记为"已失效"并通知管理员

### 10.4 未成年人数据合规

- 如需注册，建议采用家长邮箱注册机制，并遵循儿童数据最小化收集原则
- 录音文件属于敏感数据，需加密存储、限制访问权限，并在隐私政策中明确说明用途

### 10.5 免责声明

- 建议在网站底部及注册协议中明确说明视频内容来源与版权归属
- 声明模板参考："本站所有视频内容均来源于YouTube平台，版权归原创作者所有。本站仅作为教育整理与学习引导用途，不提供视频下载或再分发服务。如您认为本站内容侵犯了您的合法权益，请联系我们将及时处理。"

### 11.2 镜像站方案（前端硬编码 + 环境变量配置）

> 注：镜像站名称和可用性随时可能变化，使用前需验证。列表通过环境变量 `NEXT_PUBLIC_MIRROR_SITES` 配置，格式为逗号分隔的URL模板。

**URL 模板替换规则**：将 YouTube URL 中的 `youtube.com` 替换为镜像站域名。例如：
- 原始：`https://www.youtube.com/watch?v=abc123`
- y2u.be 镜像：`https://y2u.be/watch?v=abc123`

**前端切换逻辑**：
```
尝试 YouTube → 失败 → 尝试镜像站1 → 失败 → 尝试镜像站2 → ... → 全部失败则提示管理员
```

**常用镜像站候选**（使用前逐一验证）：

| 镜像站 | 替换规则 | 稳定性 | 备注 |
|---|---|---|---|
| y2u.be | youtube.com → y2u.be | 中等 | 较常用，支持大部分视频 |
| vidplay.me | youtube.com → vidplay.me | 待验证 | - |
| 其他 | 根据实际使用效果持续补充 | - | - |
